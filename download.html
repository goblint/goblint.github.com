Download and Installation

The latest stable version of the analyzer is available below as a source distribution. Since the analyzer is rapidly developing, we recommend using the development versions of Goblint and CIl as described on our developer pages on GitHub.

Command-line Tool

It is very simple to install goblint on a Debian-based Linux distribution.

  1. Download and extract the following archive: goblint.tgz.
  2. Follow the instructions in the README file. Basically, it should suffice to write: 
    sudo apt-get install ocaml-findlib libounit-ocaml-dev libxml-light-ocaml-dev
./build_all.sh
  3. You can test if the analyzer works by running it on the test programs; for example: 
    ./goblint tests/regression/04-mutex/01-simple_rc.c
If this works, you are ready to use the analyzer. In order to analyze larger C projects, you have to give all source files as parameter to the analyzer.

The Eclipse Plug-In

There is a preliminary version of an Eclipse plug-in available. You need to have Eclipse and the CDT installed. Then, you can obtain the plug-in through the following update site:

    http://goblint.in.tum.de/plugin

You can either set it through Help → Install New Software ...→ Add ... or by going to the update site and downloading it manually and placing it the Eclipse plug-in folder. Then, you need to indicate the location of the executible through Window → Preferences... → C/C++ → Goblin. Finally, you may need to reset the C/C++ perspective (Window → Reset Perspective ...).

When the plug-in is installed. You can call the analyzer by right-clicking in a source file and selecting "Run the Goblin" Unfortunately, this only works if you start the project as a Makefile project. Note that the plug-in feeds all C-files of a project to the analyzer irrespective of which file is active.
